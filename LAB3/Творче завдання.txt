Придумати тест на дану тему з одним питанням і чотирма варіантами відповідей та розписати чому саме такі варіанти. Тест повинен бути цікавим і повчальним, там повинен розглядатися якийсь нюанс роботи з мовою С. (1 бал).
Що виведе програма після введення символу a, якщо її код виглядає наступним чином?
#include <stdio.h>

int main() {
    char letter;
    scanf("%c", &letter);
    printf("%c", letter - 32);
    return 0;
}
Варіанти відповіді:
1) a
2) A
3) Символ нового рядка (Enter)
4) Помилка компіляції

Пояснення:
Варіант 1 (неправильний): Виведення букви a є неможливим, оскільки програма перетворює її на велику букву за допомогою операції віднімання.
Варіант 2 (правильний): Програма перетворює малу літеру на велику, оскільки віднімається значення 32 від ASCII-коду літери a, яке дає ASCII-код великої літери A.
Варіант 3 (неправильний): Виведення символу нового рядка (Enter) не відбудеться, тому що символ нового рядка не буде прочитаний до виконання операції зчитування.
Варіант 4 (неправильний): Помилка компіляції не буде, оскільки код є синтаксично правильним.

Придумати одне завдання з даної теми і повністю його розібрати. Завдання повино бути цікавим і повчальним, в ньому повинен розглядатися якийсь нюанс роботи з мовою С. (1 бал)
Завдання: Напишіть програму, яка зчитує ціле число та перевіряє, чи є воно кратним числу 5, 10 або 15.
Приклад коду:
#include <stdio.h>

int main() {
    int number;
    printf("Enter an integer: ");
    scanf("%d", &number);

    if (number % 5 == 0) {
        printf("The number is a multiple by 5.\n");
    } 
    if (number % 10 == 0) {
        printf("The number is a multiple by 10.\n");
    } 
    if (number % 15 == 0) {
        printf("The number is a multiple by 15.\n");
    } 
    if (number % 5 != 0 && number % 10 != 0 && number % 15 != 0) {
        printf("The number is not multiple by 5, 10, or 15.\n");
    }

    return 0;
}

Це завдання допомагає зрозуміти, як використовувати операцію взяття залишку для перевірки дільності, а також як правильно структурувати умови для перевірки декількох можливих варіантів.
